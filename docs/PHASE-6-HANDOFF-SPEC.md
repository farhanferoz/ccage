# Phase 6 ŌĆö Handoff + Resume Cost Interception (Spec v2)

Workflow: **Sonnet implements each subphase TDD-style; Opus reviews when subphase reports green.** Same rhythm as PLAN.md.

## Why

`claude --resume` (`-r`) and `claude --continue` (`-c`) trigger a **structural** prompt-cache miss in Claude Code, not a TTL-based one: `processSessionStartHooks('resume')` and `reorderAttachmentsForAPI` shuffle bytes at the start of `messages`, breaking cache at offset 0 every time. Cache continuity is **never preserved** across resume ŌĆö verified by GitHub issues #42309, #43657, #44045, #51764 across versions 2.1.69ŌĆō2.1.138, and by empirical inspection of session JSONLs on this machine (2-minute gap still produced a 36K-token cache-write spike).

Users pay the cache-write tax on every resume. On a 100K-token Opus session: ~$1.88 per resume. There is no Claude-Code-side knob to fix this; cache TTL is also not user-configurable (no settings.json key, env var, or CLI flag exposes the 1h-extended ttl).

ccage's role: surface the cost before launch, and offer a free, offline alternative ŌĆö a structural handoff brief extracted from the session JSONL with `jq`.

## Goals

- **A.** `ccage handoff [<session-id>]` ŌĆö produce a Markdown brief from a session JSONL using only `jq` and shell. Zero API calls. Output suitable for pasting as the first message of a fresh `claude` session.
- **B.** `-r` / `-c` cost interception in the `claude` shell function ŌĆö compute precise rewrite cost from JSONL `usage` fields and prompt user before launch.

## Non-goals

- LLM-quality narrative synthesis (`/session-handoff` from inside a warm session already covers this ŌĆö users invoke it manually).
- Daemonized idle detection or keep-alive ping.
- Fixing Claude Code's structural cache miss (Anthropic's problem).
- Auto-paste / auto-clear (too magical; brittle).

## Feature A ŌĆö `ccage handoff`

### Surface

```
ccage handoff                          # most-recent session for $PWD
ccage handoff <session-id>             # specific session (UUID, prefix-match allowed)
ccage handoff --output FILE            # explicit path; default derived from $CCAGE_HANDOFF_DIR
ccage handoff --project <abs-path>     # other project's session
ccage handoff --max-prompts N          # cap user-prompt section (default 20)
ccage handoff --stdout                 # write to stdout instead of file
```

**Default output directory:** `${CCAGE_HANDOFF_DIR:-${XDG_DATA_HOME:-$HOME/.local/share}/ccage/handoffs}`. Global flat layout ŌĆö one directory across all projects. Filenames include project slug to disambiguate: `<project-slug>-<session-prefix>-<YYYYMMDD-HHMMSS>.md`.

No `--json` flag. YAGNI; add when a consumer exists.

### Lookup

1. Project slug derived from `$PWD` (or `--project`) by `${path//\//-}`. Matches Claude Code's internal naming ŌĆö verified empirically against `~/.claude/projects/`.
2. Session directory: `$CLAUDE_CONFIG_DIR/projects/<slug>/`.
3. No `<session-id>` arg ŌåÆ most-recent by mtime (`ls -t "$DIR"/*.jsonl | head -1`).
4. Prefix `<session-id>` ŌåÆ match by leading chars; multiple matches ŌåÆ exit 2 with candidate list.

### Brief format

```markdown
# Handoff: <session-id>

**Project:** /home/ff235/dev/ccage
**Started:** 2026-05-15 09:14
**Last activity:** 2026-05-16 18:32 (4h 12m ago)
**Turns:** 47 user / 47 assistant
**Tokens billed so far:** 124,567 input ┬Ę 8,432 output ┬Ę 89,034 cache-write ┬Ę 412,890 cache-read
**Estimated cost so far:** ~$2.13 (claude-opus-4-7, 200K tier)
**Last model used:** claude-opus-4-7

## User prompts (last 20 of 47)

1. ...
2. ...
...
20. ...

_(27 earlier prompts elided ŌĆö see raw JSONL at <path> for full history)_

## Files touched (top 30 by frequency)

| Path | Read | Edit | Write |
|---|---:|---:|---:|
| /home/ff235/dev/ccage/RESUME.md | 3 | 1 | 0 |
| ... |

## Commands run (deduplicated, trivial elided)

- `git log --oneline -20`
- ...

## Last assistant turn

<verbatim text; truncated to first+last 300 words if >600 words, marked with `ŌĆ”(N words elided)ŌĆ”`>

---
Generated by `ccage handoff` from <jsonl-path> on 2026-05-16T22:30:00Z
```

### Truncation defaults

- User prompts: most-recent 20 (older counted in elision marker).
- Files-touched table: top 30 rows by `Read+Edit+Write` count.
- Bash commands: deduplicate exact-match; drop trivial (`pwd`, `ls`, `true`, `clear`). Cap at 40 unique entries.
- Last assistant turn: if >600 words, keep first 300 + last 300, insert `ŌĆ”(N words elided)ŌĆ”` between.

All thresholds tunable via env vars in a later phase if real-world use shows they're wrong.

### jq extraction primitives (`share/ccage-handoff.sh`)

| Datum | jq approach |
|---|---|
| User prompts (human only) | `select(.type=="user") \| select(.toolUseResult==null) \| select(.isMeta!=true) \| .message.content \| (string-or-text-array)` |
| Files touched | `select(.type=="assistant") \| .message.content[]? \| select(.type=="tool_use" and (.name=="Read" or .name=="Edit" or .name=="Write")) \| {op:.name, path:(.input.file_path // .input.path)}` ŌåÆ group/count |
| Bash commands | `select(.type=="assistant") \| .message.content[]? \| select(.type=="tool_use" and .name=="Bash") \| .input.command` ŌåÆ dedup, filter trivials |
| Last assistant text | `select(.type=="assistant") \| .message.content[]? \| select(.type=="text") \| .text` ŌåÆ `tail -n 1` |
| Token totals | sum across all assistant turns' `.message.usage.*` |
| Last model | last assistant record's `.message.model` |
| Timestamps | first/last record `.timestamp` |

Implementation must include test fixtures covering:
- `<system-reminder>`-wrapped synthetic user messages (exclude).
- Tool-result records (exclude from "user prompts").
- Multi-line content arrays vs string content (both render).
- Empty/malformed lines (skip, count, warn at end).

### Edge cases

- No JSONL exists ŌåÆ exit 2, message: `no sessions found for $PWD`.
- Multiple matches on prefix ŌåÆ exit 2, list candidates one per line.
- Malformed JSONL line ŌåÆ skip, increment warning count, print summary at end. Log full jq error to `$CCAGE_DIR/handoff.log`.
- Zero user prompts ŌåÆ still emit brief with placeholder section.

### Entry point

New `bin/ccage` POSIX-sh dispatcher (matches the `ccage list/doctor/prune` roadmap in RESUME.md):

```sh
case "$1" in
  handoff) shift; exec "$CCAGE_LIB/ccage-handoff.sh" "$@" ;;
  -h|--help) ... ;;
  *) echo "ccage: unknown command: $1" >&2; exit 2 ;;
esac
```

Installer drops `bin/ccage` into `$PREFIX/bin/` and verifies it's on PATH.

### "Next steps" UX after generation

Default behavior (when invoked from interactive shell):
1. Write brief to `${CCAGE_HANDOFF_DIR:-${XDG_DATA_HOME:-$HOME/.local/share}/ccage/handoffs}/<project-slug>-<session-prefix>-<YYYYMMDD-HHMMSS>.md`.
2. Print path on stdout.
3. Print next-step hint: `next: run \`claude\` (fresh session) and paste this file's contents as your first message`.
4. Clipboard auto-copy if available ŌĆö detection order: `pbcopy` (macOS) ŌåÆ `wl-copy` (Wayland) ŌåÆ `xclip` (X11) ŌåÆ `xsel` (X11 fallback). First match wins. Append `(also copied to clipboard)` to the hint when successful. Skip silently if none found. No OSC 52 escape fallback in v0 (some terminals choke on it).

With `--stdout`: print brief to stdout, no path, no hint, no clipboard. For piping/scripts.

## Feature B ŌĆö `-r` / `-c` Cost Interception

### Where it hooks

`share/claude-isolation.sh`, inside the `claude()` shell function, between `_ccage_bootstrap_dir` (current line 197) and `_ccage_pre_exec_hook` (current line 204):

```bash
_ccage_intercept_resume "$@" || return $?
```

### Detection ŌĆö three regimes

The bare-args case has to be handled differently from explicit-id, because `claude -r` without a session id may invoke Claude Code's own interactive session picker (we can't predict which session the user picks).

| Args | Behavior | Why |
|---|---|---|
| Contains `-c` or `--continue` | Intercept. Use most-recent session by mtime. | Deterministic ŌĆö Claude Code always picks most-recent. |
| Contains `--resume <id>` or `-r <id>` (next arg looks like UUID/prefix) | Intercept. Use that session id. | Deterministic ŌĆö user named the session. |
| Contains bare `-r` / `--resume` with no id (or next arg is another flag) | **Pass through, no prompt.** | Claude Code may show its own picker; we don't know which one will be picked. |

Flag-vs-positional distinction: treat the token after `-r`/`--resume` as a session id if and only if it matches `^[0-9a-f-]{6,}$` (UUID prefix shape).

### Cost computation

The empirical evidence from JSONLs shows:
- Tools+system prefix (~19K typical) hits at account level even on resume.
- The message prefix (everything else cached) gets fully re-written.
- Pre-gap `cache_read` value on the last substantive assistant turn Ōēł size of cached prefix at that moment.

Formula:

```
substantive_reads = [cache_read_input_tokens of each assistant turn where it exceeds 1000]
if substantive_reads is empty:
    skip prompt (no signal ŌĆö pass through)
peak_read = max(substantive_reads)
predicted_message_prefix = max(0, peak_read - 19000)  # subtract tools+system
predicted_rewrite_low  = predicted_message_prefix * 0.85
predicted_rewrite_high = predicted_message_prefix * 1.25
```

Honest about ┬▒25% uncertainty in the prompt: show a range, not a point.

### Pricing

Hardcoded table at `share/ccage-pricing.sh`. Per million tokens, 200K tier (1M tier costs 2├Ś ŌĆö not handled in v0, document as caveat). Cache-write at 1.25├Ś input.

```
# updated: 2026-05-16
claude-opus-4-7    input=15     cache_write=18.75
claude-sonnet-4-6  input=3      cache_write=3.75
claude-haiku-4-5   input=0.80   cache_write=1.00
```

Computed cost displayed using model from last assistant record's `.message.model`. If unknown, show both Opus and Sonnet estimates.

### Threshold ŌĆö dollar-denominated

```
CCAGE_RESUME_PROMPT_MIN_USD=0.25    # default; skip prompt below this
```

Compute back to tokens using detected model rate. Below threshold, skip prompt and pass through silently ŌĆö the cost isn't worth the friction.

### Prompt UX

```
ccage: Session 4f616b4b ┬Ę 47 turns ┬Ę last activity 4h 12m ago ┬Ę claude-opus-4-7
       Resuming will rewrite ~70K tokens (message prefix). Estimated cost: $1.10ŌĆō$1.65.
       (Claude Code's resume always misses cache; not a TTL issue.)
       [r]esume / [h]andoff / [c]ancel?
```

Implementation:

```bash
_ccage_resume_prompt() {
  local response
  read -n 1 -s response < /dev/tty
  echo "$response"
  _ccage_resume_decide "$response"
}

_ccage_resume_decide() {
  case "$1" in
    r|R|"") echo resume ;;
    h|H)    echo handoff ;;
    *)      echo cancel ;;
  esac
}
```

`_ccage_resume_decide` is a pure function ŌĆö straightforward to unit-test. The `read` wrapper is integration-tested via stub TTY.

### Branch actions

| Decision | Action |
|---|---|
| `resume` | Return 0 from interceptor; `claude()` proceeds normally. |
| `handoff` | Run `ccage handoff <session>`; print path + paste hint; return 1 (cancel claude invocation). |
| `cancel` | Print `cancelled` to stderr; return 1. |

### Env-var contract

| Var | Default | Effect |
|---|---|---|
| `CCAGE_NO_RESUME_PROMPT` | unset | If `1`, skip interception entirely. |
| `CCAGE_RESUME_PROMPT_MIN_USD` | `0.25` | Skip prompt below this estimated cost. |
| `CCAGE_RESUME_PROMPT_MODEL` | auto | Override model used for cost display. |

All reads use `${var:-default}` for `set -u` safety. Folded with the existing `set -u` fix (see ┬¦ Phase 6b commit plan).

### Edge cases

- No JSONL in expected location ŌåÆ pass through (claude will error naturally).
- `-r -p` / `--resume --print` (headless) ŌåÆ cannot prompt; pass through with one-line stderr warning.
- `[ ! -t 0 ]` (stdin not a tty) ŌåÆ pass through silently.
- `CCAGE_DISABLE=1` ŌåÆ skip interception (existing ccage opt-out semantics).
- Session id arg matches multiple sessions ŌåÆ pass through with warning (don't guess).

## Pricing data freshness

Pricing rot is inevitable. v0 mitigation:
- `share/ccage-pricing.sh` has a `# updated: YYYY-MM-DD` header.
- CHANGELOG entry on every rate refresh.
- `ccage doctor` (future phase) will warn if `pricing.sh` is >180 days old.
- No runtime fetch ŌĆö installer-time only ŌĆö to keep ccage offline-clean.

## Cache-fix sentinel (post-v0, deferred)

If Anthropic ever fixes the structural `-r` cache miss (issues #42309 / #43657 are still open as of 2026-05-16), the resume-cost prompt becomes noise.

Future `ccage doctor` should include a sentinel: run a 2-message session, `/clear`, `-r`, inspect the next assistant turn's `cache_creation_input_tokens`. If it's near-zero, print: `Claude Code resume cache appears fixed; consider \`export CCAGE_NO_RESUME_PROMPT=1\`.`

Not auto-disabling ŌĆö let the user decide.

Out of scope for Phase 6a/6b. Track as Phase 6+.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| JSONL schema changes (Anthropic renames fields) | Tests pinned to fixtures; failures are loud, not silent. Document as "best-effort, schema-dependent" in README. |
| Pricing rates drift | `# updated:` header in `ccage-pricing.sh`; CHANGELOG on refresh; future `ccage doctor` age check. |
| `-r` detection false positives | Whole-word flag match only (`-r`, `--resume`, `-c`, `--continue`); session-id distinguished by UUID-prefix regex. Tests cover edge cases. |
| Cost estimate accuracy | Show as range with ┬▒25% uncertainty; copy explicitly states "estimate"; user can disable prompt. |
| Anthropic fixes structural resume cache miss | Prompt becomes noise; doc the `CCAGE_NO_RESUME_PROMPT=1` opt-out; defer sentinel check to Phase 6+. |
| Brief paste itself uses context tokens | Truncation defaults (20 prompts, 30 files, 40 commands, 600-word assistant tail); honest about diminishing returns for huge sessions. |
| jq dep missing on minimal systems | Install-time check with clear hint; document in README Requirements. |

## Decisions log

What was considered and rejected during spec design:

- **LLM-summary handoff via `claude -p`** ŌĆö rejected because the user clarified they want no separate Claude Code invocations beyond what's in their main session flow. Free jq extraction is strictly within the constraint.
- **Anthropic API calls (Haiku) for cheap summarization** ŌĆö rejected: separate API billing, violates the no-API-call constraint.
- **`--json` output flag** ŌĆö rejected as YAGNI; add when a consumer exists.
- **Auto-paste / auto-feed brief into fresh session** ŌĆö rejected as too magical and fragile; user controls the paste.
- **Background daemon for idle detection + keep-alive ping** ŌĆö rejected: operational complexity, cost math marginal, most idle sessions never get returned to.
- **Single fixed cost in prompt vs range** ŌĆö rejected; empirical error is ┬▒20ŌĆō25%, so a range is honest and a point estimate is misleading.
- **Intercepting bare `claude -r`** ŌĆö rejected because Claude Code may show its own picker; pass through.
- **Token-denominated threshold** ŌĆö rejected; cost-of-noticing is dollar-shaped, not token-shaped.
- **Bundling `set -u` fix into a feature commit** ŌĆö rejected for git-hygiene reasons; keep as separate commit within Phase 6b.
- **Git context in handoff brief** ŌĆö rejected. ccage must not assume git usage. Users may work in non-git projects (sandboxes, scratch dirs, non-VCS workflows). The brief already lists files-touched from tool_use records; users wanting git state can run `git status` themselves in the new session. No `--with-git-context` flag at any phase.
- **Per-project handoff retention** (`$CLAUDE_CONFIG_DIR/handoffs/`) ŌĆö rejected. Global flat layout at `~/.local/share/ccage/handoffs/` enables cross-project listing and uniform `ccage prune` semantics. Filenames embed project slug for disambiguation.
- **Stdout-only handoff default** ŌĆö rejected. The `-r` interception's `[h]andoff` branch needs a file path to print, so a temp file would be required anyway. Default to writing a real file at the canonical location.
- **OSC 52 terminal-escape clipboard fallback** ŌĆö rejected for v0. Real SSH/remote utility, but some terminals (legacy iTerm modes, certain `screen` configs) break on the escape. Revisit if remote-session usage materializes.
- **Differentiated `-c` vs `-r <id>` prompt copies** ŌĆö rejected. Single template with verb swap (`Continuing most-recent session` vs `Resuming session`); identical otherwise. Same information either way.
